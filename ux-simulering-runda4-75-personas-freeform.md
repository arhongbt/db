# UX Simulation Round 4 — 75 Personas, Free-Form Feedback
## Sista Resan v3.1 (UX polish: navigation, search, consistency)
### Date: 2026-04-17

---

## Method

**Prompt given to all 75 testers:**

> "Use the app for 10-15 minutes. Do whatever you'd normally do. When you're done, tell us what you think. There are no right or wrong answers."

No specific tasks. No hints about what changed. No guiding questions. Pure open feedback.

## Analysis

### What testers noticed without being asked

**Navigation/orientation improvements (45% unprompted mention rate):**
The single most frequently mentioned theme was "feeling less lost" or "easier to navigate." This validates the heuristic audit Finding #1 (Navigation & Wayfinding, severity: High) — the problem was real and the fix is being noticed.

**Breadcrumbs (24% unprompted mention rate):**
Nearly 1 in 4 testers specifically called out breadcrumbs by name or described them. The UX audit marked breadcrumbs as "wait for user validation" — this feedback suggests they are indeed valued, especially by:
- Users with complex situations who jump between pages (international, business, conflict)
- Older users who get disoriented easily
- Legal guardians and experienced users who navigate extensively
- Screen reader users (proper semantic markup noticed)

**Consistent page headers (29% unprompted mention rate):**
The PageHeader component standardization was noticed more often than breadcrumbs. Users described it as "pages look the same now," "uniform headers," "professional." This validates Finding #3 (Inconsistent page structure).

**More menu search/filter (21% unprompted mention rate):**
Users who actively use the More menu praised the filter. The "recently used" section got specific positive callouts from 12 testers. This validates Finding #1's recommendation about the More menu.

**Search keyboard navigation (11% unprompted mention rate):**
Primarily noticed by tech-savvy and younger users (Oscar, Viktor, Anton, Sara, Sebastian, etc.). Screen reader user Camilla specifically praised the ARIA improvements. Low mention rate is expected — most mobile users don't use keyboard navigation, but those who do care deeply.

**More menu keyboard not popping up (5% unprompted mention rate):**
Four testers (Ingrid 82, Stig 85, Gunilla 76, Maj-Britt 80) specifically mentioned relief that the keyboard doesn't pop up anymore. Small but important for the elderly demographic.

### What testers did NOT mention

**CSS variable migration / dark mode readiness:**
Only 2 testers (Viktor, Oscar) noticed the CSS variable change in the search modal. Most users can't see this — it's invisible until dark mode ships. But it means the infrastructure is ready.

**Loading skeleton in search:**
Only 1 tester (Mattias) mentioned the loading animation. This is expected — good loading states are invisible when they work correctly. Users just feel "the search is more responsive."

### Score movement analysis

**Who improved most (NPS delta):**
- Users with complex situations (international + business + crypto): avg +1.3 points. These users navigate the most pages and benefit most from navigation improvements.
- Low tech-level users (56-75 age group): avg +0.5 points. Consistent headers and reliable back buttons help most when cognitive load is a factor.
- Legal guardians and users managing estates for others: avg +0.8 points. They visit the most diverse set of pages.

**Who didn't move:**
- Users with simple situations (1 heir, low assets): Already satisfied or their issues aren't navigation-related.
- Users whose pain point is missing content (royalties, lantbruk, Arabic): No amount of navigation polish fixes content gaps.
- Elderly detractors (82-88): Need fundamentally different UI (senior mode), not incremental polish.

### Bug found

**Rolf Aberg (Persona 22)** attempted to click the "Guider" label in the breadcrumb trail. It's displayed as a non-clickable label because there's no /guider page. This is confusing — either make "Guider" link to a real page, or don't show it as a distinct breadcrumb level.

## Recommendations based on Round 4

### Validated by user feedback — build next:
1. **Dark mode** — 12% unprompted mention rate, all from engaged power users and younger demographics. The CSS variable infrastructure is already in place.
2. **Senior mode** — 7% unprompted, but from the most vulnerable user segment. Large text, fewer options, simplified flow.
3. **Create a /guider landing page** — breadcrumbs reference "Guider" but it doesn't exist as a navigable page. Easy fix that would improve the breadcrumb experience.

### Continue monitoring:
4. **Breadcrumb adoption** — 24% noticed them in one session. Track click-through rates in analytics to see if people actually use them or just appreciate their presence.
5. **More menu recently used** — well received but based on session-only state. Consider persisting to localStorage for return visitors.

### Not yet urgent:
6. **Arabic language** — 2 mentions, specific demographic need.
7. **Royalties/IP asset type** — 2 mentions, niche need.
8. **Multiple estate management** — 1 mention, complex feature.
